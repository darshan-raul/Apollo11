## What's next (Stage 8: Command Module)

Stage 8 teaches **security and access control**:

- **RBAC** — `viewer` role for passengers (read their own bookings), `admin` role for flight management staff (CRUD on flights + bookings). Uses the 13 ServiceAccounts from stage 6/7.
- **SecurityContext** on all pods: `runAsNonRoot`, `readOnlyRootFilesystem`, `allowPrivilegeEscalation: false`, `capabilities.drop: [ALL]`. This is the hardened k8s security profile.
- **OPA Gatekeeper** — `K8sPSP` constraints (no privileged, no hostNetwork, etc.) as `ConstraintTemplate` + `Constraint` resources.
- **Vault** for secret management (HashiCorp Vault sidecar / Agent Injector pattern).
- **NetworkPolicies tightening** — finally enforce (in clusters with a CNI that supports it). On kind, leave them as reference.
- **Code changes**:
  - All 5 backend Dockerfiles: non-root user (USER 1001 / 65532).
  - All 5 backend Services: ServiceAccount annotation `iam.amazonaws.com/role` (placeholder for Stage 9 AWS IRSA).
  - Trace context propagation fully wired (the OTEL SDK from Stage 6 already does this; Stage 8 may add manual span creation for cross-service async events).
