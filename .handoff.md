## Critical Architecture Decisions (Must Not Revert)

### 1. Namespace labels — required for NetworkPolicy
Namespaces do NOT automatically get a `name` label in Kubernetes. The NetworkPolicy's `namespaceSelector: matchLabels: name: apollo11-apps` ONLY works because we explicitly added `labels: name: apollo11-apps` to each namespace.yaml. The `kubernetes.io/metadata.name` label is also present as the canonical well-known label.

### 2. Traefik for Ingress, Envoy for Gateway API
- Legacy Ingress: `ingressClassName: traefik` (NOT nginx)
- Gateway API: `gatewayClassName: envoy` (NOT nginx)
- Traefik annotation: `traefik.ingress.kubernetes.io/router.entrypoints: web,websecure`
- Envoy Gateway is the next-gen Gateway API controller

### 3. serviceAccountName on every deployment
All 11 deployments have `serviceAccountName: apollo11-<infra|apps|ui>` set explicitly in `spec.template.spec`. Not inherited from namespace default.

### 4. Init jobs in apollo11-apps namespace
Init jobs run in `apollo11-apps` but connect to infra DBs via FQDN:
- `pg_isready -h catalog-postgres.apollo11-infra.svc.cluster.local -p 5432`
The `.apollo11-infra.svc.cluster.local` suffix is required for cross-namespace DNS resolution.

### 5. Frontend is NodePort (not Ingress yet)
Frontend uses NodePort:30080. Stage 3 replaces this with proper Ingress for external access.
